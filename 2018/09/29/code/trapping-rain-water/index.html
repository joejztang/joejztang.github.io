42. trapping rain water

2018-09-29

description

Given n non-negative integers representing an elevation map where the width of each bar is 1, compute how much water it is able to trap after raining.

For example,
Given [0,1,0,2,1,0,1,3,2,1,2,1], return 6.

The above elevation map is represented by array [0,1,0,2,1,0,1,3,2,1,2,1]. In this case, 6 units of rain water (blue section) are being trapped. Thanks Marcos for contributing this image!

analysis1 - dp

ltr[i] represents the highest bar from 0th to ith (including ith)
rtl[i] represents the highest bar from len-1th to ith (including ith)

induction rule: res = min(ltr[i], rtl[i])-A[i]

time O(n)

space O(n)

solution1

analysis - two pointers

two pointers

leftmax and rightmax points to leftmost and rightmost bar
谁小移谁 往中间夹, (同时变换 leftmax and rightmax 所指向的值)

why can we do this? because we traverse from leftmost/rightmost to middle.
until we actually see current idx, it remains unknown, but even if unknown,
if it’s less than the less large of the two pillar, we still get water.

time O(n)

space O(1)

solution2

analysis3 - two pointer

  1. find the peak in the middle, so every pillar in either left or right will be <= the peak
  2. traverse from the leftmost and rightmost, using the largest of current to calculate the result

time O(n)

space O(1)
