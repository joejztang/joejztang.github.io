300. longest increasing subsequence

2018-10-13

description

Given an unsorted array of integers, find the length of longest increasing subsequence.

Example:

Note:

  • There may be more than one LIS combination, it is only necessary for you to return the length.
  • Your algorithm should run in O(n2) complexity.

Follow up: Could you improve it to O(n log n) time complexity?

analysis

lis[i] represents the longest increasing subsequence of first ith character (including ith)

base case: all of lis would be 1 (the least increasing subsequence will be itself)

induction rule: if (nums[i]>nums[j]) lis[i] = max(lis[i], lis[j]+1);

return max.

time O(n^2)

space O(n)

Follow up:

In order to have nlogn time complexity, we want to use another approach to solve it.
We can try to build a subsequence that contains LIS, and return the length of it.

When the element greater than previous, we can append it to the partial LIS, it will contribute to the final result;
however when the element is less than previous, we need to find the first greatest element in previous sub list that less than this element and set the one to current element after that. (use bisect_left)
To find it, we can use binary search, because remember when building the partial LIS it will keep acsending.

Another whim need to mention is that DON’T use stack and pop out the elements when the element is less than its previous elements.
By using stack, we only recorded on possibility at the moment, and it’s easily losing other possibilities during the pop.

One of the example that could be wrong is [0, 1, 0, 3, 2, 3]. When we met the second 0, it will eliminate 1 from the stack, and we lose that whole possibility later on.

time O(nlogn)
space O(n)
