kth smallest number in sorted matrix

description

Given a matrix of size N x M. For each row the elements are sorted in ascending order, and for each column the elements are also sorted in ascending order. Find the Kth smallest number in it.

Assumptions

  • the matrix is not null, N > 0 and M > 0
    K > 0 and K <= N * M

Examples

the 5th smallest number is 4
the 8th smallest number is 6

dijkstra solution

time

O(klogk)

space

O(MN)

heap solution

问 kth 最大或最小的题目都可以用 heap 来做. 1.这道题维护最大堆, 建堆 klogk 复杂度. 2. 诶个和堆顶比较, 如果比堆顶值小就 poll, 然后 offer, 每个 poll&offer 的复杂度是 logk 复杂度, 剩下 (m*n-k) 个元素, 所以就是 (mn-k)(logk) 的复杂度.

time

(mn-k)(logk)+klogk = O(mnlogk)

space

O(k)

solution binary search

上面是最容易想的方法. 这道题有两个性质:

  • 行有序
  • 列有序

应该可以用 binary search 来做. 这道题的 bs 方法我来分析一下.

假如有如下的 matrix:

left=1 right=4 midVal=2

这个 midVal 是我们猜的第 k 小的值.

然后数这个数列中比 midval 小的有多少个, 记为 num;

如果 num 大于 k, 那么在小的这一群里找
num小于 k 则在大的那一群里找.

这道题难想的地方是怎么找到这个 num, 来说说这个方程的思路.
首先,recall 这个 matrix 性质

从列最大的 row (用 i 记录)的第一个数和我们猜的 midVal 比
if matrix[i][j] < midVal: 那么i 行及 i 行以上整个列都小于 midVal, 并且 j 向后串一个; 如果matrix[i][j] > midVal: 那么, 保持现有的 j, i 向上串一个.

你可能会有这样的疑问, 这样会不会漏掉一些数? 答案是不会.

matrix[0][0]是整个matrix 最小值, matrix[n-1][n-1]是整个 matrix 最大值. 当某个值matrix[m][n]大于 midVal 的时候, 这个值得右边的值matrix[m][n+1]一定大于 midVal. 当某个值matrix[m][n]小于 midVal 的时候, 这个值的上边的所有值matrix[*][n]一定小于 midVal.

我们来模拟一下这个程序.

k = 2

left=1, right=10, mid=5. num=3, 此时 right 更新为5

left=1, right=5, mid=3. num=3, 此时 right 更新为3

left=1, right=3, mid=2. num=2, 此时 right 更新为2

left=1, right=2, 不满足 left<right-1 条件, 停止.

检查1和2. 其中一个是所要的解.
