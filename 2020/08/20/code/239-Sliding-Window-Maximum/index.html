Description

Given an array nums, there is a sliding window of size k which is moving from the very left of the array to the very right. You can only see the k numbers in the window. Each time the sliding window moves right by one position. Return the max sliding window.

Follow up: Could you solve it in linear time?

Example:

Constraints:

  • 1 <= nums.length <= 10^5
  • -10^4 <= nums[i] <= 10^4
  • 1 <= k <= nums.length

Analysis

I am using a java generic structure TreeMap which represents a BST. So every time I query the top, it will give the max.

However, time complexity of that would be O(nlogk), space complexity O(k).

Solution

Follow up

The idea is similar to the solution I came up, just instead of using BST to find the top, we need another data structure to quickly find the max.

We are going to use a deque.

This deque will record the index of max element inside current window. How do we achieve that? We compare window last with current element, if the element found by windows index last is less or equal to current element, it means windows index last is not the largest in current window anymore, pop it up. In this way, current window will record max elements on the top and potential max following.

Time O(n)
Space O(k)
